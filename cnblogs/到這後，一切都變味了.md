
很多東西來了這裏都變了，VPN、eSIM、Apple、Google、Harmony 這些是你可能已經知道的產品或公司，如果你在國內工作，也應該能感受到這種變化，就我所在的軟件行業，底層大量的國產化之後，你的技術棧也會大量國產化：

> 這兩年中美脫鉤，我從業的公司，也正在把大量的依賴遷移到國產，這是大趨勢。我們知道，所謂的國產，可能就是 Fork -> Replace -> Rebuild -> Compiler，時刻兼容上游修改的同時，添加更多國內客製化的需求進去，比如各家輿情審查，內容和諧審查等，至少我覺的就是這麼回事。
>
> 而我總是跟着國外的教程文檔，一步一步過河，踩坑，產品也只用國外，儘量不碰國產。如果我再在國內面試時，又被面試官揪起來問，爲什麼沒有用金蝶、用友、東方通、碼雲、麒麟、龍芯、容器雲的產品，我會不會再次破防崩潰？
>
> 因爲產品對口，完全沒有適應成本，這我也能理解，但是我更害怕的是，如果我長期浸泡在國內市場，我整個人的技術棧和經驗，也會完全跟着國產化，靠自己跟着前沿大佬學習，實在是有太大的阻力，我也不能像北歐那些高福利國家那樣，拿着低保去寫代碼，暫時我還是得在這裏謀生，所以除了寄出更多的時間來滾社區外，還真一點辦法都沒有。

這是一方面，還有另一方面，你每日用僅剩的精力，僅剩的時間做出來的項目也未必會被人賞識，有些人甚至會認爲你沒有全心全意工作，沒有把全部精力投入到工作中去。

能怎麼辦呢？

好在找工作是個雙向選擇的事情，就算考僞裝真的進入到這個系統，後面一定還有更加難受的事情等着你，儘早遠離這些「傻 \*」是一種智慧。

但這應該還不是最壞的，大環境不斷變壞的現在，我們完全有理由假想未來有一天會更糟，那個時候，可能簡歷至少要也是雙份的，在根據語言領域切分之後，可能還需要再把簡歷裏面 GitHub 項目切分掉，避免開放出自己的 GitHub 地址。這種變化跟我兩年前找工作時大相徑庭 [^edu-project]。

[^edu-project]: 因爲學生時代很少有實習經歷，我的項目基本全在 GitHub 上。

說不難受是假的，但好像真的一點辦法都沒有。
